## Пример 2:
Этот пример демонстрирует сбор данных с нескольких веб-сайтов с использованием веб-скрейпинга.

Допустим, у вас есть задача собрать информацию с нескольких веб-сайтов. Вы хотите сканировать каждый сайт параллельно, чтобы ускорить сбор данных, но вы не хотите перегружать сетевые ресурсы или серверы сайтов. В этом случае ограничение параллелизма может быть полезным.

Вы можете использовать "Bounded Parallelism" для ограничения количества одновременных HTTP-запросов к сайтам. Например, установив ограничение в 5 одновременных запросов, вы гарантируете, что ваш скрейпер не отправляет более 5 запросов одновременно. Это позволит:

- Собирать данные с нескольких сайтов одновременно, ускоряя процесс.
- Предотвращать перегрузку серверов веб-сайтов, поскольку вы не отправляете слишком много запросов одновременно.
- Эффективно использовать ресурсы, так как не создаются слишком много параллельных горутин, что может вызвать избыточное потребление памяти.
